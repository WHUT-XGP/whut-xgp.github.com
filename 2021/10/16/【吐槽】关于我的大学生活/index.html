【吐槽】关于我的大学生活

October 16, 2021 1317

高考毕业为了读计算机专业读了一个普通211， 在大学里，课程内容很是让人费解和困惑。 身为计算机专业，大一上学期不允许带电脑，会被查且鼓励举报，查到会被要求把电脑邮寄回家，只有成为班干部才能够获得所谓的特批权利。 这个学校会花一整年的时间来教一门编程语言，但是却又只讲语法，而且是最基础的语法内容，稍微宽泛、高级和困难一些的概念都要求自己去掌握和了解，比如不讲STL。 身为计算机专业，反而编程语言考试没有机考，而且题目大多没有什么实际价值和意义，甚至有很多语言未有标准定义，依赖编译器自己实现的输入输出题，大题还要求手写代码。 花半年的时间来让学生跟着视频抄一个终端版本的网吧计费管理系统，内容非常简单，只涉及到一些最基础的文件IO操作和一些时间相关的处理。讽刺的是，整个年级能独立按照需求文档完成的人，寥寥无几，绝大多数人依赖抄袭学长、室友和照着视频抄写。 计算机学院有个所谓的专业分流，分流后处于不同校区，相距几十公里，受不同的人管辖。 两边学习的画风完全不一样。 随便有一些行政相关的事情要处理的时候，都需要异地跑来跑去，徒劳增加管理成本和行政效率。 我分流后选择来到了另一个校区，这个校区大而散乱，课程安排和生活安排有冲突，上课需要走接近 3 公里，半小时路程，每天有很大一部分时间花在路上。可能是自己的缘故，总觉得这样无意义的赶路，是在徒然耗费学生精力和学习热情，影响效率。 在另一个校区，情况未必得到好转，很多课程同样囿于形式。大一已经花了一整年去了解什么是编程语言，学习过了 C++，我理解应该算是已经入门计算机世界，然后大二又是从语法讲起学习 Java，老旧设备可能不支持投屏，所以老师并不带自己的办公电脑，每次带个 u 盘先装 10 分钟 JDK 再开始上课，从语法继续讲起，安排大批量的低端任务和代码编写，写几十页的实验报告，个人不熟练 office，对我来说，往往处理 word 的时间大于我为这个实验写的代码量和思考量。 对于 Java 对应的应用场景一概不提，教一些过时老套的 GUI 代码编写。教一些没有技术含量的代码，生怕学生毕不了业，实验课则直接变抄写题。 这个大学有个很有意思的点，基本上所有的课程和实验都是前辈的学长学姐做过的内容，很多时候几乎是完全不改动任何一个任务要求。 饶是如此，也有大批学生需要借助其他人的代码才能蒙混过关，这里我没有别的意思，因为我自己也有这样的时候。 专业课讲的内容，除了少数课程，其他大多数知识点，都是一些可以追溯到 20 年前的过时知识，很多人说大学本科就是进行基础理论和知识图谱的搭建，我承认这一点，经典的理论和有用的符合实际的知识我向来没意见，但是这不代表可以用一些明显过时和不符合事实的东西去教授给学生。毕竟我校某位老师有在课堂上念过：“ JavaScript 是 Java 的成熟版本 ”这样一句话，给当时初学前端的我带来了很严重的心理阴影和打击，因为这件事，我一度怀疑自己在这个学校学到的零碎知识点里，能有多少是正确的，如果我自己缺失了判断力和自信，我会被灌输多少错误的概念，毕竟不是所有的错误我都能精确明白正确与否。 暂且吐槽到这吧，毕竟我还没毕业。
